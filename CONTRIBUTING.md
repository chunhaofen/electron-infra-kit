# Contributing to Electron Infra Kit

Welcome to the Electron Infra Kit community! We're excited that you're interested in contributing to our project. This guide will help you understand how to get involved and make meaningful contributions.

## ü§ù Ways to Contribute

There are many ways to contribute to the Electron Infra Kit, including:

### 1. Code Contributions

- **Bug fixes** - Fix issues reported in the [GitHub Issues](https://github.com/chunhaofen/electron-infra-kit/issues)
- **New features** - Implement new features that align with the project's roadmap
- **Performance optimizations** - Improve the performance of existing code
- **Code refactoring** - Improve code quality and maintainability

### 2. Documentation

- **API documentation** - Improve or expand the API documentation
- **User guides** - Create or update tutorials and guides
- **Examples** - Add new examples or improve existing ones
- **Translations** - Translate documentation to other languages

### 3. Testing

- **Unit tests** - Write unit tests to increase code coverage
- **Integration tests** - Create integration tests for complex features
- **Manual testing** - Test the library and report bugs or issues

### 4. Community Support

- **Answer questions** - Help other users on GitHub Issues, Stack Overflow, and Reddit
- **Share knowledge** - Write blog posts or create videos about using the library
- **Promote the project** - Share the project with others who might find it useful

## üìã Contribution Process

1. **Find an issue** - Look for issues labeled `good first issue` or `help wanted` if you're new to the project
2. **Fork the repository** - Create your own fork of the project
3. **Create a branch** - Create a new branch for your contribution
4. **Make changes** - Implement your changes following the code style guide
5. **Test your changes** - Run the test suite to ensure your changes don't break anything
6. **Submit a PR** - Create a pull request with a clear description of your changes
7. **Review process** - Wait for maintainers to review your PR and address any feedback
8. **Merge** - Once approved, your PR will be merged into the main branch

## üìù Code Style Guide

- Follow the existing code style and patterns
- Use TypeScript for all new code
- Write clear, concise comments
- Ensure all tests pass before submitting a PR
- Maintain or increase code coverage

## üèÜ Contribution Rewards and Recognition

We value all contributions and want to recognize the efforts of our community members:

### 1. Contributor Badges

- **First Contributor** - For your first accepted PR
- **Code Contributor** - For code-related contributions (5+ PRs)
- **Documentation Champion** - For documentation contributions (3+ PRs)
- **Bug Hunter** - For reporting 5+ verified bugs
- **Community Supporter** - For answering 10+ questions on GitHub or Stack Overflow

### 2. Contributors List

All contributors will be listed in the project's README.md file under the "Contributors" section.

### 3. Maintainer Opportunities

Active contributors who consistently provide high-quality contributions may be invited to become project maintainers.

### 4. Swag

From time to time, we may send swag (stickers, t-shirts, etc.) to our most active contributors as a token of appreciation.
